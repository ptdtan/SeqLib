1 /* udc - url data cache - a caching system that keeps blocks of data fetched from URLs in
2  * sparse local files for quick use the next time the data is needed.
3  *
4  * This cache is enormously simplified by there being no local _write_ to the cache,
5  * just reads.
6  *
7  * The overall strategy of the implementation is to have a root cache directory
8  * with a subdir for each file being cached. The directory for a single cached file
9  * contains two files - "bitmap" and "sparseData" that contains information on which
10  * parts of the URL are cached and the actual cached data respectively. The subdirectory name
11  * associated with the file is constructed from the URL in a straightforward manner.
12  * http://genome.ucsc.edu/cgi-bin/hgGateway
13  * gets mapped to:
14  * rootCacheDir/http/genome.ucsc.edu/cgi-bin/hgGateway/
15  * The URL protocol is the first directory under the root, and the remainder of the
16  * URL, with some necessary escaping, is used to define the rest of the cache directory
17  * structure, with each '/' after the protocol line translating into another directory
18  * level.
19  *
20  * The bitmap file contains time stamp and size data as well as an array with one bit
21  * for each block of the file that has been fetched. Currently the block size is 8K. */
22
