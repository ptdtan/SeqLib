1 /* hacTree - Hierarchical Agglomerative Clustering a list of inputs into a binary tree */
2 /*
3  * This clustering algorithm can be used for problems with high
4  * dimensionality, such as clustering sequences by similarity. It can
5  * be used for problems with low dimensionality too, such as
6  * clustering a sequence of numbers (1-dimensional: points along the
7  * number line), but there are more efficient algorithms for problems
8  * with low dimensionality.
9  *
10  * In order to use this, you need to provide functions that operate on a
11  * pair of input items (and extra data pointer):
12  * 1. a distance function that takes two input items and returns a number,
13  * small for short distances/high similarity and large for long
14  * distances/low similarity
15  * 2. a merging function that takes two input items and returns a new item
16  * that represents both of them. Therefore the data structure that
17  * represents an input item must also be capable of representing a
18  * cluster of input items. Usually this is done by combining values
19  * from input items into a new value that will compare properly with
20  * other items or clusters in the distance function.
21  *
22  * Then pass in an slList of items, pointers to those two functions, and
23  * a pointer to data used in your distance and merging functions (or NULL).
24  * You get back a pointer to the root node of a binary tree structure
25  * where each leaf node contains one of your items and each node above
26  * that contains a cluster. The root node contains the cluster of all items.
27  *
28  * If a significant proportion of input items are identical to each other,
29  * you can also pass in a comparison function that will be used to sort the
30  * items before clustering. After sorting, adjacent identical items will
31  * be pre-clustered in order to reduce the number of inputs to the main
32  * clustering step.
33  *
34  * kent/src/lib/tests/hacTreeTest.c contains a couple simple examples.
35  *
36  * To get the top k clusters, perform a breadth-first,
37  * lowest-distance-child-first search of the tree to find the top k
38  * nodes/clusters with the smallest distances from the root node/cluster.
39  * [It would be nice for this module to provide that function, if/when a need for it arises.]
40  */
