Constructor & Destructor Documentation

~BamReader ( )
inline

Destroy a BamReader and close all connections to the BAMs.

Calling the destructor will take care of all of the C-style dealloc calls required within HTSlib to close a BAM or SAM file.

BamHeader Header ( ) const

Return a copy of the header to the first file.

Note
The object returned is a copy of the BamHeader, but this does not actually copy the actual header contents. Header contents are stored in a shared_ptr, and so the new returned BamHeader have a copy of the shared_ptr that will point to the originally alloced raw header data.
