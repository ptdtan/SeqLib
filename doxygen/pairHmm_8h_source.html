10 /* Mommy coding scheme - this is how one cell in the dynamic programming table
11  * points to it's parent (mommy) cell. Since these tables are really big,
12  * rather than use a simple pointer costing four bytes, we use a encoding
13  * scheme that requires only one byte.
14  *
15  * Bits 0-4 the "hidden" state of the mommy. Lets us have 32 hidden states.
16  * currently only using 7.
17  * Bit 5 whether or not mommy is in previous cell in query
18  * Bit 6 whether or not mommy is in previous cell in target
19  * Bit 7 set during trace back for cells along optimal path
20  *
21  * Since the query and target advancing bits (5 and 6) are never both zero,
22  * it is safe to use the value of all-bits-zero as an indicator of
23  * no mommy. */
24 
25 /* Compress state, query, and target offset into one byte. */
26 #define phmmPackMommy(stateIx, qOff, tOff) ((UBYTE)((stateIx) + ((-(qOff))<<5) + ((-(tOff))<<6)))
