139  /* This is the type of a client provided function to save an alignment.
140  * The parameters are:
141  * chromName - name of target (aka genomic or database) sequence.
142  * chromSize - size of target sequence.
143  * chromOffset - offset of genoSequence in target.
144  * ffAli - alignment with pointers into tSeq/qSeq or in
145  * translated target case, into t3Hash.
146  * tSeq - part of target sequence in normal case. In translated
147  * target case look at t3Hash instead.
148  * t3Hash - used only in translated target case. A hash keyed by
149  * target sequence name with values *lists* of trans3 structures.
150  * This hash can be searched to find both the translated and
151  * untranslated versions of the bits of the target that are in
152  * memory. (You can assume at this point all parts needed for
153  * output are indeed in memory.)
154  * qSeq - query sequence (this isn't segmented at all).
155  * isRc - True if query is reverse complemented.
156  * stringency - ffCdna, etc. I'm hoping to move this elsewhere.
157  * minMatch - minimum score to output. Also should be moved elsewhere.
158  * outputData - custom data for specific output function.
159  * The interface is a bit complex - partly from the demands of translated
160  * output, and partly from trying not to have the entire target sequence in
161  * memory.
162  */
