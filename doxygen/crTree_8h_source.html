1 /* crTree chromosome r tree. This module creates and uses a disk-based index that can find items
2  * that overlap with a chromosome range - something of the form chrN:start-end - with a
3  * minimum of disk access. It is implemented with a combination of bPlusTrees and r-trees.
4  * The items being indexed can overlap with each other.
5  *
6  * There's two main sides to using this module - creating an index, and using it.
7  *
8  * The first step of index creation is actually to insure that the file being indexed
9  * is ordered by chromosome,start,end. For a .bed file you can insure this
10  * with the command:
11  * sort -k1,1 -k2,2n -k3,3n unsorted.bed > sorted.bed
12  * Note that the the chromosome field is sorted alphabetically and the start and end
13  * fields are sorted numerically.
14  *
15  * Once this is done then the index creation program scans the input file, and
16  * makes a list of crTreeItems, one for each item in the file. A crTreeItem just
17  * contains the chromosome range and file offset for an item. In the process of scanning,
18  * build up a hash containing the chromosome names (using hashStoreName) rather than allocating
19  * a separate string for each chromosome name in each item. Pass the list and the
20  * hash to crTreeFileCreate.
21  *
22  * Using an index is done in two steps. First you open the index with crTreeFileOpen,
23  * and then you use crTreeFindOverlappingBlocks to find parts of the file the overlap
24  * with your query range. The result of a crTreeFindOverlappingBlocks call is a list
25  * of regions in the file. These regions typically include some non-overlapping items
26  * as well. It is up to the caller to parse through the resulting region list to
27  * convert it from just bytes on disk into the memory data structure. During this
28  * parsing you should ignore items that don't overlap your range of interest.
29  *
30  * The programs crTreeIndexBed and crTreeSearchBed create and search a crTree index
31  * for a bed file, and are useful examples to view for other programs that want to
32  * use the crTree system. */
